# Cling Design Composition Patterns

## 总原则

一张图里：

- 一个主角色
- 一个主情绪
- 一圈信息碎片
- 一个能记住的主姿态
- 一个主文字区（可选，但最多一个）

不是把内容拆步骤，而是把“这个人在面对这些信息时的状态”画出来。

## 比例适配规则

同一张图换比例时，不能只裁画布，必须换排法。

### `16:9`

- 适合横向展开
- 适合流程、桌面、左右对照、前后转化
- 节点可以横排，但每一段都要留呼吸感

### `3:4`

- 适合封面和社媒卡图
- 人物更容易成为主角
- 结构要从“横向讲清楚”改成“纵向抓重点”
- 主标题区和人物区先锁死，再决定碎片往上还是往下漂

### `1:1`

- 适合方图封面和头像延展
- 画面天然更挤，所以碎片数量默认减 15%-25%
- 更适合一个主角色 + 2-4 个信息点
- 不适合硬塞完整流程

### `9:16`

- 适合全屏短视频首屏、story 图
- 禁止照搬横版流程
- 结构必须上下分层
- 通常做成：主标题 / 主角色 / 关键信息块 三段式

## 模式 1：中心角色 + 环绕碎片

最常用。

适合：

- 内容总结
- 情绪表达
- 思考状态
- 观点海报

做法：

- 角色在中间或略偏一侧，但一定要大
- 周围是 4-8 个漂浮小元素
- 小元素可以是气泡、纸片、卡片、符号
- 角色可以压到画面边缘，制造封面感

## 模式 2：角色 + 桌面

适合：

- 准备
- 改稿
- 整理内容
- 工位脑暴

做法：

- 角色坐在桌边或趴在桌前
- 桌上有摊开的书、文档、纸堆
- 周围再飘一圈信息碎片
- 桌面形状用直角块面，不要圆角卡片

## 模式 3：角色 + 对话冲击

适合：

- “所以呢”
- 被打断
- 已读不回
- 对面不听了

做法：

- 角色是受力点
- 一个更大的气泡/词块/手机提示占据另一侧
- 让视觉重心体现“被压住”
- 词块和提示框优先直角，不要圆润可爱

## 模式 4：角色 + 转化装置

适合：

- 把乱内容变清楚
- 从散到整
- 从长文档到一图流

做法：

- 角色作为操作员
- 左边混乱，右边清爽
- 机器可以怪一点，但不要工业流程图
- 左右结构可以更像“辣妹在收拾烂摊子”，不是教程

## 模式 5：小人嵌进流程

适合：

- 内容生产流程
- 方法链路
- 多阶段叙事
- 同一人物贯穿不同节点

做法：

- 流程是主角，人物是固定资产
- 同一个角色可以缩成 3-5 个小分身，但必须保持同一造型
- 每个节点里的人物只是动作线索，不抢流程结构
- 节点 props 只保留最低必要量，避免教程页感
- 如果人物变小，画面噪音也要同步减少

## 信息碎片规则

信息碎片可以用：

- 小卡片
- 聊天气泡
- 文件页
- 书本
- 标点
- 小符号块

不要把它们排成整齐九宫格。  
要像围着角色飘，而不是排版演示页。

额外硬规则：

- 可读文字碎片最多 2-3 块
- 如果已经有主标题，其他碎片尽量不要再写完整句子
- 其余碎片优先用色块、线条、符号、纸张边缘、半遮挡内容来制造氛围

## 构图提醒

- 角色先被看见
- 碎片后被看见
- 主标题如果存在，应当第二眼才抓到
- 颜色块只做节奏点
- 留白要够，别把四周塞满

## 禁止套路

这套 skill 不该走：

- 冷海报剪影
- 纯几何强构图
- 无角色的抽象概念海报
- 硬信息图
- 教程页
- 圆角 UI 卡片排排坐
- 满屏小字贴纸
