# Cling Design Composition Patterns

## 总原则

一张图里：

- 一个大标题区
- 一个主角色
- 一个主情绪或主判断
- 一圈同主题信息贴片
- 一个能记住的主姿态
- 一个主文字区（必有，而且通常要大）

不是企业流程图，也不是纯角色海报，而是：

- **标题先抓人**
- **角色承载情绪**
- **贴片补充判断**

## 比例适配规则

同一张图换比例时，不能只裁画布，必须换排法。

### `16:9`

- 适合横向展开
- 适合流程、桌面、左右对照、前后转化
- 最稳结构是：左侧大标题 + 中右主角色 + 四周贴片
- 节点可以横排，但不要变企业模板

### `3:4`

- 适合封面和社媒卡图
- 人物更容易成为主角
- 结构要从“横向讲清楚”改成“纵向抓重点”
- 最稳结构是：上左标题区 + 中间主角色 + 四周贴片
- 主标题区和人物区先锁死，再决定碎片往上还是往下漂

### `1:1`

- 适合方图封面和头像延展
- 画面天然更挤，所以碎片数量默认减 15%-25%
- 更适合一个主角色 + 2-4 个信息点
- 不适合硬塞完整流程

### `9:16`

- 适合全屏短视频首屏、story 图
- 禁止照搬横版流程
- 结构必须上下分层
- 通常做成：主标题 / 主角色 / 关键信息块 三段式

## 模式 1：大标题 + 中心角色 + 环绕贴片

最常用。

适合：

- 内容总结
- 情绪表达
- 思考状态
- 观点内容卡

做法：

- 标题占据一块强区域，通常在左上或左侧
- 角色在中间或略偏一侧，但一定要大
- 周围是 4-8 个贴片或漂浮小元素
- 小元素可以是气泡、纸片、卡片、符号
- 角色可以压到画面边缘，制造封面感

## 模式 2：标题 + 角色 + 桌面

适合：

- 准备
- 改稿
- 整理内容
- 工位脑暴

做法：

- 标题和桌面同场存在
- 角色坐在桌边或趴在桌前
- 桌上有摊开的书、文档、纸堆
- 周围再飘一圈信息贴片
- 桌面形状用直角块面，不要圆角卡片

## 模式 3：标题 + 角色 + 对话冲击

适合：

- “所以呢”
- 被打断
- 已读不回
- 对面不听了

做法：

- 标题必须先打中
- 角色是受力点
- 一个更大的气泡/词块/手机提示占据另一侧
- 让视觉重心体现“被压住”
- 词块和提示框优先直角，不要圆润可爱

## 模式 4：标题 + 角色 + 转化装置

适合：

- 把乱内容变清楚
- 从散到整
- 从长文档到一图流

做法：

- 标题先说结论
- 角色作为操作员
- 左边混乱，右边清爽
- 机器可以怪一点，但不要工业流程图
- 左右结构可以更像“辣妹在收拾烂摊子”，不是教程

## 模式 5：小人嵌进流程

适合：

- 内容生产流程
- 方法链路
- 多阶段叙事
- 同一人物贯穿不同节点

做法：

- 流程是主角，人物是固定资产
- 同一个角色可以缩成 3-5 个小分身，但必须保持同一造型
- 每个节点里的人物只是动作线索，不抢流程结构
- 节点 props 只保留最低必要量，避免教程页感
- 如果人物变小，画面噪音也要同步减少
- 标题区必须继续像杂志跨页的大字标题，不要缩成说明标签
- 节点之间允许有拼贴感、纸片感、错位感，避免排成标准教材流程
- 整体要像“内容卡式流程图”或“杂志感方法图”，不是正式说明书

## 信息贴片规则

贴片可以用：

- 小卡片
- 聊天气泡
- 文件页
- 书本
- 标点
- 小符号块
- checklist
- 手机消息框
- 会议记录纸
- 手写便条

不要把它们排成整齐九宫格。  
要像围着角色和标题区形成“信息场”，不是排版演示页。

额外硬规则：

- 大标题只有一个
- 贴片默认 4-8 块
- 其中可读文字贴片可以更多，但必须都是短句、短项或短泡泡
- 所有贴片必须服务同一个主题，不准东一块西一块
- 不要满屏都是长整句

## 构图提醒

- 标题先被看见
- 角色第二眼被看见
- 贴片第三眼被读到
- 颜色块只做节奏点
- 留白要够，但不能空到没有内容感
- 流程图也必须保留版面设计感和杂志感，不能只剩步骤清楚

## 禁止套路

这套 skill 不该走：

- 冷海报剪影
- 纯几何强构图
- 无角色的抽象概念海报
- 硬信息图
- 教程页
- 圆角 UI 卡片排排坐
- 满屏随机小字贴纸
