# Cling Design Operating Rules

这份不是灵感草稿，是已经验证后的最终工作规程。

## 一句话定义

`cling-design` 现在只服务一条路：

- `approx`：正文 / 流程 / 插图 / 情绪图 / 观点配图

一句话：

- **让意思先成立，再让气质成立。**

## 适用任务

- 正文配图
- 流程图
- 情绪图
- 观点插图
- 工作状态图
- 多动作、多节点、多场景表达

### 典型任务

- “根据这段文字出一组正文图”
- “给我一张内容生产流程图”
- “做一张表达工作状态的插图”
- “按这张参考图的味道出一张正文配图”

### 不适合的任务

- 需要固定人物一模一样的封面
- 人设封面
- 头像延展
- 大字主钩子的 strict cover

这些已经不属于当前 skill 的活跃工作流。

## approx 的硬规则

`approx` 的目标不是“同一个人完全不变”，而是：

- 气质一致
- 场景自然
- 信息表达成立

### 人物规则

- 可以近似扩展
- 保住短 bob、圆墨镜、黑高领、冷感
- 不要求像素级同一个人

### 结构规则

- 优先动作、场景、节点
- 允许多人物小分身
- 允许三场景叙事
- 允许流程表达，但不要掉进低配流程框模板

## 比例默认值

- 流程/结构/横向表达优先 `16:9`
- 普通社媒正文图优先 `3:4`
- 方图按 `1:1`
- 全屏竖图按 `9:16`

比例变了，构图也必须重排，不能只硬裁。

## 最常见的错法

### 错法 1：把正文图也要求“人物一模一样”

结果会变成：

- 既不自然
- 也不够表达
- 还会不断漂人

### 错法 2：把 approx 做成低配流程框模板

结果会变成：

- 像 PPT 占位图
- 人物和信息脱节
- 没有 cling-design 的角色感和纸面感

## 交付判断

先问自己：

- 这张能不能先把意思讲清，再让人觉得有味道
- 场景、动作、信息是不是在一个画面系统里
- 人物是不是在帮内容说话，而不是单独抢戏

这套 skill 只有守住这个边界，才不会再反复跑偏。
